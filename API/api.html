O que endpoint? E API?

Endpoint

Ponto final ou local final que uma requisição vai chegar, isso significa que qualquer lugar na internet é um endpoint.

Normalmente se assume que o termo endpoint se refere a um endereço de uma API.

API

Application Programming Interface: Interface de Programação de Aplicações.

Interface: Tudo o que vemos hoje é uma interface.

    Exemplo:

  • Página web do github
  • Página web da globo
  • Página web do google
  • Página web da meta
  • Painel de um carro
  • Painel de uma moto

    • Tudo o que nós vemos de inicío é uma interface.

Se não estamos mexendo nos detalhes de como as coisas foram feitas ou cavucando a implementação, estamos usando uma interface.

Abstração: Evitar mexer nos detalhes de implementação para usar um componente, um model, função ou método. Para isso nos comunicamos com as interfaces, as interfaces são as bordas das funcionalidades e responsabilidades. A falta das interfaces faz o código ser macarrônico, você olha pro código e não consegue saber onde cada responsabilidade começa ou termina. As interfaces servem para usar algo de forma mais fácil, sem entender como aquilo foi feito.

Interfaces programáveis: Enquanto as páginas servem para agradar os humanos, as APIs servem para que as máquinas tenham um acesso programático (mais direto, com mais previsibilidade) as informações.

As interfaces web trazem mais ruído e tendem a mudar ao longo do tempo. A mesma interface programável (API), pode servir a diferentes interfaces "para humanos".

Alguns tipos de interface:

TUI: Text-based User Interface (Interface de Usuário baseado em Texto) é o que usamos dentro do terminal.

GUI: Graphical User Interface (Interface de Usuário Gráfica) é o que a maioria dos usuários usam dentro do navegador.

Diferentes tipos de interface podem ser criadas usando a mesma API. Clients diferentes que se comunicam via HTTP podem se comunicar com o mesmo servidor em HTTP.

curl: Abreviação de Client URL, uma ferramente que pede (Request) e a que recebe (Response) algo que foi requerido.

Um comando curl é simples assim: curl http://localhost:3000/api/status, ele verifica o que foi pedido e o que foi recebido.

Para ver em detalhes usar o comando --verbose ou -v (curl http://localhost:3000/api/status -v)

E é exatamente isso que navegador usa para mostrar as páginas web, ele faz uma request dessa mesma forma e entende quando a response é um html que ele deve formatar para o usuário.
